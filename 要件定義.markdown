# プロジェクト要件定義書: ANIME TIER MAKER (Refined)

## 1. プロジェクト概要 & ビジョン
* **アプリ名**: ANIME TIER MAKER
* **コンセプト**: **「ただ画像を並べる」を超えた、究極のアニメ特化型Tier作成ツール**。
* **既存ソリューション(TierMaker等)との差別化**:
    * **× 既存**: 画像のみの羅列。作品名が不明。検索不可。新アニメ追加にテンプレート更新が必要。
    * **○ 本アプリ**: Jikan API連携による**「生きたデータ」**の活用。
        * **検索性**: 作品名（日/英）、時期、ジャンルで即座に検索・追加可能。
        * **情報量**: ホバーで作品詳細（制作会社、放送年、ジャンル）を表示。
        * **鮮度**: 常に最新のアニメデータベースからリストを作成可能。
        * **柔軟性**: 「2024年冬アニメ」などの括りで一括インポートが可能（予定）。

## 2. 技術スタック (Strict Mode)
* **Framework**: Next.js 14 (App Router)
* **Language**: TypeScript
* **State Management**: **Zustand**
* **Styling**: Tailwind CSS, shadcn/ui
* **Drag & Drop**: @dnd-kit/core, @dnd-kit/sortable
* **API**: Jikan API v4
* **Backend**: Supabase (Auth, DB)

## 3. UI/UX デザイン原則
* **「作品」としての扱い**: 単なる正方形の画像ではなく、ポスターアート（縦長）としての美しさを尊重する。
* **直感的な検索と整理**:
    * 検索画面（Search Sidebar/Drawer）と整理画面（Tier Canvas）のシームレスな連携。
    * ドラッグ＆ドロップ時の明確なフィードバック（ドロップ領域のハイライト、アニメーション）。
* **情報のレイヤー化**:
    * **Level 1 (一覧)**: 画像のみ、または画像＋タイトル（設定で切替）。
    * **Level 2 (ホバー)**: タイトル、放送年、ジャンルなどのメタデータ表示。
    * **Level 3 (クリック)**: 詳細モーダル（将来的に実装）。

## 4. 機能要件詳細 (Updated)

### A. インテリジェント・サーチ & インベントリ
* **検索機能の強化**:
    * キーワード検索（日/英対応、Debounce）。
    * **[New] シーズンフィルター**: 「2024 Winter」「Top Airing」などのプリセットボタン。
    * **[New] バッチ追加**: 検索結果の「すべてをUnwatchedに追加」機能。
* **表示モード**:
    * カバー画像（縦長 2:3）を基本とする。
    * 既存の「正方形クロップ」強制は行わず、ユーザーが「Fit」か「Fill」か選べるようにする（現状はCover）。

### B. Tier Canvas (編集エリア)
* **行（Tier Row）の表現**:
    * **ヘッダー**: S, A, B... のラベル。クリックで色変更、テキスト編集。
    * **コンテンツエリア**: グリッド配置ではなく、Flex Wrapによる自然な配置。
* **カード表示オプション**:
    * **[New] テキストオーバーレイ**: 画像の下部、または上に半透明でタイトルを表示するトグルスイッチ。
* **Unwatchedエリア**:
    * Tier表の一部としてではなく、**「待機場所（Dock）」**として画面下部または別枠に常設することも検討（現状はTierの一つとして実装中だが、UI改善の余地あり）。

### C. データ管理 & エクスポート
* **画像生成**:
    * `html-to-image` を使用。
    * エクスポート時は「タイトル表示ON/OFF」などの表示状態をそのまま反映。
    * **[New] テキストリスト出力**: 画像だけでなく、Sランクの作品名リストなどをテキスト/JSONでコピーできる機能。
* **保存 (Supabase)**:
    * ユーザーごとのTier表保存。
    * 一般公開（Public）設定と共有URL発行。
